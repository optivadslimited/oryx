01 — Project Overview & Goals
You are building the complete digital presence for Oryx Eyewear — a premium Moroccan eyewear brand. The website must feel like walking into a luxury boutique online: refined, minimal, intentional. Every pixel communicates quality and quiet confidence.

What You Are Building
A public-facing e-commerce website (storefront)
A secure, private Admin Panel (CMS) for the brand owner to manage everything
A backend API connecting both

Core Objectives
Sell glasses online: Morocco (MAD) and worldwide (EUR) with a clear order/inquiry form
Build desire: Every page must create emotional want before functional need
Admin control: The brand owner must be able to log in and control 100% of site content without touching code
Mobile-first: 60%+ of visitors will be on mobile — design for mobile first, desktop second
Performance: Fast load times are non-negotiable for conversion on Moroccan mobile networks

The reference site is hillseyewear.com — study it carefully before starting. Adopt its scroll behavior, product presentation, editorial photography framing, and minimalist navigation style. Do not copy it — interpret it through the Oryx brand identity.

02 — Design Inspiration & Reference Sites
Primary Reference: Hills Eyewear (hillseyewear.com)
This is the closest benchmark. Study and replicate the following specific behaviors and patterns:
Navigation: Minimal fixed top bar, logo centered or left-aligned, no visual clutter, hamburger menu or sparse links only
Hero Section: Full-screen editorial photography. The product is secondary to the mood. Large slow-fade or parallax image. A single line of copy maximum.
Product Grid: Clean square/rectangular product cards. Product name, price only. Hover reveals second image or colorway selector.
Product Detail Page (PDP): Large image left (or stacked on mobile). Sticky product info panel right. Smooth image gallery transitions.
Typography: Serif display font for headings. Clean sans-serif for body. High contrast. Generous white space.
Scroll Behavior: Smooth, deliberate. Sections fade or slide in on scroll. No jarring jumps.
Color Palette: Near-white or off-white background. Deep navy or black text. Gold or warm accent. Product photos on clean neutral backdrops.
Footer: Minimal. Logo, links, Instagram icon. Single-column on mobile.

Secondary References
Ray-Ban (ray-ban.com): Study the product filter UX, the virtual try-on teaser concept, and how they handle colorway switching on PDP
Gentle Monster (gentlemonster.com): Editorial art direction, homepage as a visual statement, not a product grid
Akila (akilaeyewear.com): Clean DTC layout, how they use brand story pages alongside product pages

What Oryx Should NOT Look Like
No generic Shopify out-of-the-box themes
No banner-heavy layouts with promotional urgency banners
No overcrowded navbars with dropdowns
No cheap stock photography or lifestyle images that feel unrelated
No auto-playing video with sound

05 — Frontend — Page-by-Page Specifications
5.1 — Global Layout & Navigation
Navigation Bar
Fixed at top. Does NOT hide on scroll (unlike most sites) — always visible.
Height: 64px desktop / 56px mobile
Background: --color-cream with a very subtle bottom border (1px --color-light-grey)
Left: ORYX wordmark in Cormorant Garamond 600, 22px, --color-black. Clicking returns to homepage.
Center (desktop): navigation links — SHOP, STORY, CONTACT — DM Sans 400, 13px, uppercase, letter-spaced. Hover: --color-gold underline slides in.
Right: Cart icon (bag icon, not a cart), shows item count badge if items in cart.
Mobile: Hamburger icon right. Clicking opens full-screen overlay menu — dark background (--color-navy), large links centered, ORYX logo top-left.

5.2 — Homepage
Section 1: Hero
Full viewport height (100vh) on desktop. 85vh on mobile.
Full-bleed editorial photography. Dark, moody, high-contrast. Person wearing frames. NOT a product shot on white background.
Overlay: very subtle dark gradient from bottom (rgba(0,0,0,0) to rgba(0,0,0,0.35))
Text: centered or left-aligned. ONE headline maximum. Example: 'Worn with intention.' — Cormorant Garamond 300 italic, 80px desktop / 44px mobile, white.
CTA Button below: 'DISCOVER THE COLLECTION' — secondary ghost button, white border, white text
Scroll indicator: subtle animated down-arrow at bottom center
Hero image is set via Admin Panel → Content → Hero Image. Must support image swap without code changes.

Section 2: Featured Products
3-column grid on desktop. 2-column on tablet. 1-column on mobile.
Section heading: 'THE COLLECTION' — left-aligned, H2 style
Each card: square product image (1:1 ratio), product name below, price below that
Hover on desktop: second image fades in (front/side view swap). Subtle card lift (box-shadow increase).
'VIEW ALL' ghost link bottom-right of section

Section 3: Brand Statement
Full-width section on --color-navy background
Single centered large quote in Cormorant Garamond italic: 'I know who I am. I don't need validation.'
White text, 52px desktop / 30px mobile
Small gold divider line above the quote (2px, 60px wide, centered)

Section 4: Brand Story Teaser
Two-column layout: left 60% is a portrait editorial photograph. Right 40% has text.
Text: heading 'Born in Morocco.' + 2–3 paragraph body copy + 'READ OUR STORY' CTA
Image fades in on scroll (IntersectionObserver)

Section 5: Instagram Feed
Grid of 6 latest Instagram images (fetched via Instagram Basic Display API or Behold.so embed)
Heading: '@oryxeyewear' with Instagram icon
Each image links to Instagram post

Section 6: Footer
Three columns on desktop. Stacked on mobile.
Column 1: ORYX logo + 'Born in Morocco. Built for everywhere.'
Column 2: Nav links — Shop, Story, Contact, Size Guide, FAQ
Column 3: Email newsletter signup form (name + email, minimal, one-line) + Instagram link
Bottom bar: © 2026 Oryx Eyewear · Privacy Policy · Terms

5.3 — Shop Page (/shop)
Header: full-width short banner image. 'THE COLLECTION' heading overlaid.
Below header: filter/sort bar — Filter by: Frame | Color | Price | Sort: Newest / Price ↑ ↓
Product grid: 3 columns desktop, 2 columns tablet, 1 column mobile
Product card elements: primary image, hover to show secondary image, frame name, colorway name, price in MAD (with EUR price in smaller text below), 'ADD TO BAG' on hover
Out of stock products: still show with 'SOLD OUT' badge. Do not hide them.
Pagination OR infinite scroll — preference for pagination (cleaner, better for conversion tracking)

5.4 — Product Detail Page (PDP) — /shop/[slug]
This is the most important page. It does the selling. Every element must be considered.
Layout
Desktop: 50/50 split. Left: image gallery. Right: sticky product details panel.
Mobile: stacked. Images on top, info below.
Left — Image Gallery
Primary large image. Thumbnail strip below (or vertical strip on the side on desktop).
Clicking thumbnail swaps main image with smooth crossfade.
Swipe gesture support on mobile.
Lightbox on click: full-screen image overlay with zoom capability.
Support minimum 5 images per product (front, side, three-quarter, model wearing, detail close-up).
Right — Product Info Panel (Sticky on Scroll)
Product Name — H1, Cormorant Garamond 600, 42px
Colorway Name — DM Sans, 14px, uppercase, --color-mid-grey
Price — DM Sans 500, 26px, --color-black. If on sale: show original price struck through in grey
Colorway Selector — visual swatches (circle or small square), clicking swaps images and updates URL
Size Guide link — opens modal, not new page
Product Description — 2–4 sentences. DM Sans 400, 16px.
Details accordion: Frame Material | Lens Type | Temple Length | Bridge Width | Lens Width | Origin
ORDER NOW / INQUIRE button — Primary CTA, full-width on mobile, fixed-width on desktop
Below button: 'Free shipping in Morocco on orders over 500 MAD' — small, gold text
Stock indicator: 'Only X left' if stock < 5. 'In Stock' if > 5. 'Sold Out' if 0.

5.5 — Order / Checkout Form
IMPORTANT: This is a FORM-BASED checkout, not a standard cart checkout. Customers fill in their details and Oryx manually confirms/processes orders. This is standard for early-stage DTC brands in Morocco. The form submits to the backend, creates an order record, sends email notification to admin, and sends confirmation email to customer.
Form Fields
Full Name: Required | Text input
Phone Number: Required | Tel input | Auto-detect Morocco country code, but allow international
Email Address: Required | Email input | Used for order confirmation
City: Required | Dropdown: Casablanca, Rabat, Marrakech, Tangier, Other
Full Delivery Address: Required if city is not 'Other' | Textarea
Country: Required | Defaults to Morocco | Other options available for international
Product: Auto-filled from PDP. Shows product name, colorway, price. Not editable by customer.
Quantity: Default 1, max 3
Special Notes: Optional | Textarea | 'Any special instructions or questions'
Payment Method: Radio select: Cash on Delivery (COD) | Bank Transfer | Other (specify)
Form Behavior
Client-side validation before submit. Show inline errors in red below each field.
On submit: shows loading state on button. On success: redirect to /order/[id] confirmation page.
On error: show inline error message, do not clear form.
Admin receives email notification with full order details.
Customer receives order confirmation email with summary.

5.6 — Brand Story Page (/story)
Long editorial scroll. Think magazine article, not corporate About page.
Large hero image top. Heading: 'Born in Morocco. Built for everywhere.'
Alternating text/image sections as user scrolls.
Pull quote sections: large gold italic text on dark background.
End with a product CTA: 'Discover the first collection →'

5.7 — Animations & Motion
Page load: Fade in from slightly below (translateY 20px → 0, opacity 0 → 1, duration 600ms, ease-out). Stagger children by 80ms.
Scroll reveal: IntersectionObserver triggers fade-in + translateY animation for each section. Threshold: 0.15.
Image hover: transform: scale(1.03) on product card images. transition: 400ms ease.
Button hover: Background color transition 250ms ease. No jump.
Mobile nav: Overlay slides in from right or fades in. Backdrop blur behind.
Product image swap: Crossfade (opacity transition 300ms)
DO NOT use jarring slide-in animations or dramatic transforms. The brand is calm and precise. Motion should be felt, not seen.

06 — Admin Panel — Full Specification
The admin panel is a completely separate section of the site, accessible only at /admin. It requires authentication. It should feel clean and functional — not consumer-facing. Think Shopify admin, but simpler.

6.1 — Authentication
Single admin account (the brand owner). No multi-user system needed for V1.
Login: email + password. JWT token stored in HttpOnly cookie.
Session expires after 24 hours of inactivity.
Forgot password: sends reset link to registered admin email.
Login page: minimal, centered card. Oryx logo. No public access to any /admin route without valid session.

Total Products: Active / Total

Last 10 orders. Columns: Order ID, Customer Name, Product, Total, Status, Date
Status options (colored badges): Pending / Confirmed / Shipped / Delivered / Cancelled

Image Upload Section (Critical)
The image upload section must be robust and user-friendly. The admin should never need to use FTP or external tools to add product images.
Drag-and-drop upload zone — supports multiple files at once
Click to browse also supported
Accepted formats: JPG, PNG, WebP

After upload, images appear as thumbnails in a sortable grid (drag to reorder)

07 — Backend & API Specification
Architecture
RESTful API (or GraphQL if developer prefers — REST recommended for simplicity)
All API routes prefixed with /api/v1/
Authentication via JWT. Public routes (GET products, POST orders) need no token. Admin routes require Authorization: Bearer [token] header.
All responses return JSON. Error responses include an error.message field.

Stock Management Rules
When an order is submitted (status: pending), do NOT deduct stock yet
When admin sets order to 'Confirmed', deduct stock from the variant
If admin cancels an order that was confirmed, restore stock
Display stock warnings on PDP: 'Only X left' when stock ≤ 5
'Sold Out' state: CTA button changes to 'NOTIFY ME' — captures email for restock notification

11 — Hosting, Performance & SEO
Performance Requirements
Google Lighthouse score target: Performance ≥ 90, Accessibility ≥ 90, SEO ≥ 95
LCP (Largest Contentful Paint): < 2.5s on 4G mobile
All product images: WebP format, max 200KB per image at display size
Hero images: max 400KB. Use Next.js <Image> with priority={true} on hero
Fonts: preload display font subset. Use font-display: swap.
Lazy load all below-fold images
No blocking third-party scripts on initial load

SEO Requirements
Every page has unique <title> and <meta description> — managed via admin content panel
Product pages: structured data (JSON-LD) for Product schema including price, availability, image
Sitemap.xml auto-generated and submitted to Google Search Console
robots.txt: disallow /admin, allow everything else
All images have descriptive alt text (admin-managed)
Clean URL slugs — no query parameters in public URLs
Hreflang tags if bilingual (Arabic/French) support is added later
Open Graph tags on all pages for social sharing

Security
All admin routes protected by JWT middleware
Rate limiting on order submission and login endpoints (prevent spam/brute force)
Input sanitization on all form fields before database insert
Environment variables for all secrets (never hardcode API keys)
HTTPS enforced. Redirect all HTTP to HTTPS.
CORS configured to allow only the production domain
